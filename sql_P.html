3. 주요 기능
출결 상황 기록 Procedure

접근 번호에 따라 교육생, 관리자로 구별하고 접근 시간에 따라 출결현황을 출결 기록 테이블에 갱신합니다.
또한 관리자가 특정 시간에 접근하면 출근 혹은 퇴근 입력이 되지 않은 교육생들의 출결 테이블에 결석으로 일괄 update를 진행하고 해당 기록 테이블을 전체 초기화를 진행합니다.
또한 중도탈락자를 가려내는 procedure를 호출합니다.

중도 탈락자 구별 Procedure

출결 테이블에 있는 출석 이외의 상황의 개수와 각각의 점수를 계산하여 일정 점수 이상이 되면 해당 학생의 레코드를 찾아 중도탈락 여부의 필드에 중도탈락을 갱신합니다.

출결 상황 기록 Trigger

출결 기록 테이블에 insert나 update문이 실행되면 출근, 지각, 조퇴 등의 출결 상태를 구별하여 출결 테이블에 insert를 진행하고 출결 상태에 따른 알림을 줍니다.

출결 시 중도 탈락자, 수료과정 구별 Trigger

출결 테이블에 insert가 실행되기 전에 해당 교육생의 레코드에 접근하여 중도탈락 여부, 수료과정의 기간을 탐색하여 중도탈락자와 수료과정이 끝났거나 아직 시작하지 않은 교육생은 에러를 발생, 메시지를 출력하여 출결 테이블에서의 입력을 거부합니다.

학생 정보 조회 Procedure

조회하고 싶은 학생의 이름을 입력받아 해당 학생의 레코드 정보를 가공하여 출력합니다.

4. 느낀 점
  • 이번 프로젝트 진행하면서 RDBMS에 대한 이해와 자신감이 생기게 되었습니다.
  • 하나의 Procedure에 여러 개의 Trigger와 Procedure을 작동하도록 만들어 차곡차곡 설정한 PL/SQL들이 발생하는 것을 보며 재미와 뿌듯함을 느꼈습니다.
